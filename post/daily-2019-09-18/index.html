每日集问-2019-09-18

Expression #1 of SELECT list is not in GROUP BY

数据库出现Expression #1 of SELECT list is not in GROUP BY 报错

在写代码的过程中，最近遇到一个问题，就是在执行 SQL 的时候一直报错，提示是Expression #1 of SELECT list is not in GROUP BY 。

由于之前都没遇到过这种问题，于是去查了一下资料。

首先导致问题的原因是 mysql 5.7.5 之后版本默认启用了 only_full_group_by SQL 模式，所以MySQL会不识别选择列表、条件或顺序列表引用的查询。也就是说，如果你 group by 的字段不在 select 中，那么就无法进行 group by。

解决方法是：

  • 登陆到服务器的mysql中
  • 输入SHOW SESSION VARIABLES; ， 查看sql_mode中是否有ONLY_FULL_GROUP_BY;属性
  • 如果有的话执行set session sql_mode='STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION';
  • 然后在查看一下是否已修改成功
  • 同样还有一个地方，SHOW GLOBAL VARIABLES;
  • set global sql_mode='STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION';
  • 如果是laravel的话，本地测试还有一种方法，就是 config/database.php这个配置 strict => false就可以了
